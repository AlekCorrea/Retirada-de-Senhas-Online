# Sistema de Senhas Online

Este repositório reúne os materiais acadêmicos, protótipos e arquivos do projeto. Use este documento como um guia para localizar cada elemento.

## Documentação e requisitos

- [Engenharia.md](Engenharia.md): requisitos e informações de engenharia do projeto.
- [Pre Projeto Aplicado v2.docx](Pre%20Projeto%20Aplicado%20v2.docx): documento do pré-projeto aplicado.
- [api.md](api.md): referência dos endpoints da API.
- [AGENTS.md](AGENTS.md): briefing de orientação para agentes de IA que atuarem no repositório.

## Protótipos e telas de referência

Os arquivos exportados ou usados como referência do Figma estão na pasta [`figma/`](figma/).

- [`figma/adm/`](figma/adm/): referências da área administrativa.
- [`figma/atendente/`](figma/atendente/): referência da área de atendimento.
- [`figma/usuario/`](figma/usuario/): referências das telas de usuário e senhas.

## Código-fonte

- [`frontend/`](frontend/): arquivos da aplicação web.
- [`backend/`](backend/): arquivos do servidor e da API.
- [`mobile/`](mobile/): arquivos da aplicação móvel.
- [`database/`](database/): arquivos relacionados ao banco de dados.
- [`nginx/`](nginx/): configuração do proxy web.

## Ambiente do projeto

- [docker-compose.yml](docker-compose.yml): arquivo de orquestração dos serviços do ambiente.
- [`frontend/Dockerfile`](frontend/Dockerfile), [`backend/Dockerfile`](backend/Dockerfile), [`mobile/Dockerfile`](mobile/Dockerfile) e [`nginx/Dockerfile`](nginx/Dockerfile): arquivos de imagem dos respectivos serviços.

## Como navegar

Abra os arquivos `.md` diretamente no editor ou no GitHub. Para o arquivo `.docx`, use Microsoft Word, LibreOffice Writer ou outro leitor compatível. Os materiais na pasta `figma/` podem ser abertos no editor como arquivos de referência; quando houver um link compartilhado do Figma, registre-o nesta seção para facilitar o acesso da equipe.
